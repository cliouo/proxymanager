Non-Mihomo output adapters remain out of scope unless code review proves an
existing product promise or a shared semantic layer requires them.

## Intentional product boundary

The supported output remains Mihomo YAML. Cross-core probes are evidence for
where semantics diverge, not a promise to generate Xray, sing-box, v2ray-core,
or legacy Clash configurations. A new output target would require an explicit
semantic model and an adapter; protocol parsers must not grow target switches.

No unsupported URI scheme is registered merely because a Mihomo type with a
similar name exists. Conversely, structured provider YAML may carry a fixed
Mihomo type that has no URI parser, provided the node passes the local
fixed-version registry and structural validator. This validator intentionally
uses stricter scalar types than Mihomo's weak decoder, but it is not a complete
reimplementation of every uncommon constructor; fixed-binary fixtures remain
the final audit oracle.

The final matrix's four finite partial rows are VMess `host`, `path`, nested
path `ed`, and VLESS `pcs`. Its only two additional whole-matrix partials are
the structural SS AEAD-2022 form distinction and the version-scoped AnyTLS
fragment. They are explicit compatibility/evidence boundaries, not unknown
behavior or silent omission. Every former wildcard, unknown query, UUID,
Hysteria port-set, HTTPUpgrade, and XHTTP trust-boundary path is now classified
as a validated mapping or an explicit rejection.
