Mihomo types without an accepted URI parser are listed in
[remaining-gaps.md](remaining-gaps.md); Mihomo support alone does not justify
inventing a URI scheme.

Across every family, unknown query/JSON keys and duplicate keys reject. The two
intentional metadata omissions are explicitly validated: SSR `group` has no
fixed-Mihomo field, and VMess schema `v` must equal `2` before being omitted.
Neither is classified as a silent drop.

The flat WireGuard URI dialect also rejects `remote-dns-resolve`: fixed Mihomo
requires it to be paired with a non-empty structured `dns[]`, which this URI
contract deliberately cannot encode. Structured provider YAML can use the pair
after every nameserver passes the conservative fixed-target grammar.
